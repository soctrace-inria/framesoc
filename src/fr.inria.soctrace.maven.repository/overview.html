This Developer Guide provides an overall description of Framesoc software architecture, and a detailed API documentation automatically generated from source code comments.

Framesoc is structured as a set of Eclipse plugins. Each plugin plugin.name contains all the packages named plugin.name.*.

The basic block of Framesoc is the soctrace library, which provides a software API to easily interact with the databases where traces are stored. The relational schema of these databases is available here . Note that the tables TRACE, TRACE_TYPE, TRACE_PARAM, TRACE_PARAM_TYPE and TOOL are physically stored in a single SystemDB. All the other tables are replicated for each trace and stored in different TraceDBs.

The soctrace library is composed by the by the following plugins.

fr.inria.soctrace.lib.model
- It provides an object oriented representation of the DB schema.
fr.inria.soctrace.lib.storage
- It provides basic functionalities for interacting with the DBs (e.g., create a new DB, open an existing one, save model elements in the DB, perform SQL queries, etc.).
fr.inria.soctrace.lib.query
- It provides classes to perform generic queries abstracting from SQL, and some utility classes to iterate over trace events, compute distributions, and interact with {@link fr.inria.soctrace.lib.model.EventProducer} hierarchy.
fr.inria.soctrace.lib.search
- It provides a simplified interface to perform predefined requests to the databases.
fr.inria.soctrace.lib.utils
- It provides utility classes to help the development of other Framesoc modules and client applications (e.g., manage DB identifiers, manage the configuration file, measuring times, ensure path portability over different OSs, etc.).

On top of the soctrace library, there are the framesoc plugins, which provide the core Framesoc mechanisms and the graphical user interface.

fr.inria.soctrace.framesoc.core
- It provides the extension point for adding new tools to Framesoc, some utility classes to help the creation of new tools, a Publish-Subscribe mechanism for decoupled module communication, and some management functionalities (e.g., register a tool, delete a Trace, etc.).
fr.inria.soctrace.framesoc.ui
- It provides all the classes supporting the Framesoc Eclipse perspective (e.g., views, graphical objects, graphical data model, etc.), and the extension point to add new Framesoc analysis views. The following Framesoc *.ui.* plugins extend this extension point to provide their analysis views.
fr.inria.soctrace.framesoc.ui.histogram
- It provides a view to visualize the trace event density histogram.
fr.inria.soctrace.framesoc.ui.piechart
- It provides a view to visualize a statistics pie chart.
fr.inria.soctrace.framesoc.ui.eventtable
- It provides a view to visualize trace events in an interactive table viewer.
